## Commands

- **Start server:** `npm start` (runs compiled `dist/server.js`, default port 3000)
- **Install deps:** `npm install`
- **Run tests:** `npm test` (vitest, runs all `tests/**/*.test.ts`)
- **Run tests in watch mode:** `npx vitest`
- **Type check:** `npx tsc --noEmit`
- **Build:** `npm run build` (compiles to `dist/`)
- **Seed data:** `npm run seed` (fetches from PokeAPI, writes `data/pokemon.json`)

## Architecture

Express + EJS server-rendered Pokedex app showing Gen 1–3 Pokemon (IDs 1–386). Written in TypeScript, compiled to `dist/` for production.

**Data flow:** `src/index.ts` → `src/routes/pokemon.ts` → `src/services/pokeapi.ts` → `data/pokemon.json`

- **src/index.ts** — Entry point. Sets up Express with EJS views and static file serving. Starts immediately (no cache warming).
- **src/services/pokeapi.ts** — Loads `data/pokemon.json` at module init via `fs.readFileSync`. All lookups are synchronous in-memory array operations behind async interfaces.
- **scripts/seed.ts** — Standalone script (`npm run seed`) that fetches all Pokemon from PokeAPI and writes `data/pokemon.json`. Run this to refresh the data.
- **data/pokemon.json** — Pre-built JSON with all 386 Pokemon including stats, abilities, flavor text, and moves. Generated by the seed script.
- **src/routes/pokemon.ts** — Factory function `createRouter(service?)` returning an Express router. Two routes: `GET /` (card grid) and `GET /pokemon/:id` (detail page). Accepts optional injected service for testing.
- **views/** — EJS templates. `index.ejs` renders the card grid, `detail.ejs` renders individual Pokemon with moves and stats. Shared layout via `partials/header.ejs` and `partials/footer.ejs`.
- **public/js/filter.js** — Client-side search filtering by Pokemon name using `data-name` attributes on cards.
- **public/css/styles.css** — All styling in one file. Type badge colors use `.type-{name}` classes (e.g., `.type-fire`).

## TypeScript Config

- `tsconfig.json` includes `server.ts` and `src/**/*.ts`, excludes `node_modules`, `dist`, `public`, `tests`, and `coverage`.
- Tests use vitest with ts support via `vitest.config.ts`.

## Key Details

- Requires Node 18+ (seed script uses native `fetch`).
- Three production dependencies: `express`, `ejs`, and `dotenv`.
- Sprites are hotlinked from PokeAPI's GitHub (`official-artwork/{id}.png`), not stored locally.
- Pokemon data is pre-built in `data/pokemon.json` — no runtime API calls.
